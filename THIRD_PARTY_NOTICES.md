# Third-Party Notices

RookDuel Avikal depends on third-party open-source software. This document summarizes notice obligations for source and packaged distributions. It is not a substitute for reviewing each dependency license before release.

## JavaScript and Electron

- Root JavaScript dependencies are declared in `package.json`.
- Frontend dependencies are declared in `frontend/package.json`.
- Electron packaged builds include Electron and Chromium license material generated by Electron tooling.

## Python Backend

- Python runtime dependencies are declared in `backend/requirements.txt`.
- Build-only dependencies are declared in `backend/requirements-build.txt`.
- CLI packaging dependencies are declared in `backend/requirements-cli.txt`.

## Rust Native Module

- Rust dependencies are declared in `backend/native/avikal_backend_native/Cargo.toml`.
- Packaged releases must preserve applicable Rust crate license notices.

## OpenSSL PQC Runtime

- Avikal PQC features use an OpenSSL 3.5+ runtime when configured or bundled.
- OpenSSL 3.x is licensed under Apache-2.0.
- Packaged runtime artifacts must include OpenSSL license and notice files.
- Runtime installation scripts should reject PQC runtime artifacts missing required license files.

## Distribution Policy

- Avikal source code is licensed under Apache-2.0.
- Third-party components remain under their own licenses.
- Packaged releases must include required third-party license files and notices.
- Production packaging generates `THIRD_PARTY_NOTICES.generated.md` from the locked npm, Python, and Rust dependency graphs.
- The generated inventory is bundled into the desktop distribution as `THIRD_PARTY_NOTICES.md`.
- Release builds fail if the inventory cannot be generated.
